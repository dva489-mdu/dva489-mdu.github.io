Excursion: Content Security Policy

In My first web application and in My first social media app we met two variants of XSS and saw how to protect against them using various forms of sanitization. While it is important to strive to remove all vulnerabilities in an application it is also very demanding, in particular as the application grows.

Content Security Policy (CSP) is a security mechanism that can be used to prevent the exploitation of injection attacks by, e.g., constraining valid script sources and preventing unsafe inline scripts.

Your task is to use CSP to protect both My first web application and My first social media app using CSP. Base your experimentation on the unpatched versions and show that your policy, indeed, protects the application from the attacks. Writing CSP policies for large web application is not an easy task as illustrated by CSP Is Dead, Long Live CSP! On the Insecurity of Whitelists and the Future of Content Security Policy. Follow the guidelines in the Content Security Policy Cheat Sheet.

Excursion Result

The intended result of this excustion is two CSP policies and an attack report that shows how the policies protect from the reflected and stored XSS attacks.
